Stock SBA Example

Introduction

The stock SBA example demonstrates a common Space-Based Architecture application containing the following components.

Data processor, data feeder, persistent space, mirror service.

This example demonstrates the following

·       Running a processor with an embedded space that is a node in a cluster as a Processing Unit.

·       Running a feeder that connects remotely to the space as a Processing Unit.

·       Running a write-behind persistency service (mirror service) as a Processing Unit.

·       Controlling the lifecycle of a Processing Unit in the Service Grid using the GigaSpaces UI.

·       Partitioning a processing task over a cluster.

·       Persisting objects from the space using a nHibernate - based External Data Source adaptor and the mirror service.

Structure

The example contains 5 projects:

·       GigaSpaces.Examples.StockSba.Commons: contains the entities project used in the example and other common functionalities.

·       GigaSpaces.Examples.StockSba.Feeder: writes the Fills remotely to the space. Its purpose is to simulate the data flowing into the system from other services/applications.

·       GigaSpaces.Examples.StockSba.Mirror: defines the mirror service as a Processing Unit.

·       GigaSpaces.Examples.StockSba.Processor: contains the processing logic implementation.

·       GigaSpaces.Examples.StockSba.PuDebugExecuter: a debugger that runs a Processing Unit lifecycle locally (without using the Service Grid), and is used to locally debug the Processing Units.

Prerequisites

1)     Download the MySql database, see: MySQL 5.0 Community Server, Without installer is the recommended version. Extract the database to “D:\MySQL”. It can also be extracted anywhere on the hard disk, this requires an update to the database location on the database-related BAT files (startMySqlPersistencyDB.bat, MySqlCreateDbAndTables.bat, see instructions below).

2)     Download the MySql .NET connector. See Connector/Net download, the Windows Binaries, no installer (ZIP) is the recommended version. Extract the file anywhere on the disk. Copy the MySql.Data.dll from the \bin directory to the <GigaSpaces Root >\Bin directory.

Building the Example

  1. Browse to the example directory (<GigaSpaces Root>\Examples\StockSba).

2.     Double-click the Build.bat file to compile the example. This script also copied the deployment files to the deploy directory.

Opening .NET Solution with Microsoft Visual Studio

1.     Double-click the GigaSpaces.Examples.StockSba.sln file.

2.     Choose Build > Build Solution menu. The project files and dependencies are created and copied to the release/ directory.

Running the Example

1.     The example must be compiled before running (see Building the Example).

2.     Start the database.

·        If its the first time you are running the example and the database and tables have not been created yet, go to <GigaSpaces Root>\Examples\StockSba\tools and run the MySqlCreateDbAndTables.bat file. If the database installation is not located in “D:\MySQL”, edit the .bat file and update the DBLocation VAR in the file to the correct installation path.

·        To run the database, double-click <GigaSpaces Root>\Examples\StockSba\startMySqlPersistencyDB.bat. If the database installation is not located in “D:\MySQL”, edit the .bat file and update the DBLocation VAR in the file to the correct installation path.

3.     Run the GSM, two GSCs and GigaSpaces UI: go to <GigaSpaces Root>\Bin and run Gs-agent.exe and run Gs-ui.exe.

4.     Use the GigaSpaces UI to deploy and run the Processing Units. Deploying all of the services on a partitioned cluster requires a large amount of memory and creates heavy load on the CPU. Therefore, it is recommended to split the example to more than a single machine. For instructions, see the Stock SBA Example documentation.

5.     The recommended order to run the Processing Units is:

1.     stockmirror  - cluster schema: <none>

2.     stockprocessor  - partitioned cluster schema.

3.     stockfeeder - cluster schema: <none>
